# 🌳 Разведение плодовых деревьев для новичков

Сайт-путеводитель по 8 плодовым деревьям: Яблоня «Мелба», Груша «Лада», Вишня «Молодёжная», Слива «Заречная ранняя», Абрикос «Краснощёкий», Персик «Киевский ранний», Черешня «Ипуть», Инжир «Брунсвик».

Статьи для каждого дерева генерируются языковой моделью через [ProxyAPI](https://proxyapi.ru) — не жёсткий шаблон, а живой, уникальный текст под каждый запрос с разной эмоциональной окраской.

## Функции

- **Главная страница** — сетка из 8 карточек с эмодзи, названием, описанием и кнопкой
- **Статьи** — генерируются LLM по запросу: вступление, описание, пошаговая посадка, советы по уходу, заглушка изображения
- **Сохранение в PDF** — кнопка «📄 Сохранить в PDF» на каждой статье использует html2pdf.js (текст статьи без UI-элементов, A4, поля 1 см)
- **Анимация появления** — карточки плавно выезжают при скролле (Intersection Observer)
- **Адаптивность** — мобильные 1 колонка, планшеты 2, десктоп 3
- **Публикация в Telegram** — кнопка «📢 Опубликовать в Telegram» отправляет статью в вашу группу через Telegram Bot API

## Установка

1. **Клонировать репозиторий:**
   ```powershell
   git clone https://github.com/av4518ai-code/trees_site.git
   cd trees_site
   ```

2. **Установить зависимости:**
   ```powershell
   pip install flask python-dotenv openai requests
   ```

3. **Создать файл `.env` в корне проекта:**
   ```
   API_KEY=ваш_ключ_от_proxyapi
   MODEL=gpt-5.5

   TELEGRAM_BOT_TOKEN=ваш_токен_бота
   TELEGRAM_CHAT_ID=id_вашей_группы
   ```

   Ключ ProxyAPI можно получить на https://proxyapi.ru → Ключи API.
   Поддерживаемые модели: `gpt-5.5`, `gpt-5.4-mini`, `gemini-3.5-flash`, `claude-opus-4-7` и др.

4. **Запустить:**
   ```powershell
   python app.py
   ```

5. **Открыть в браузере:** http://localhost:5000

## Подключение Telegram (пошаговая инструкция)

Чтобы кнопка «📢 Опубликовать в Telegram» заработала, нужно создать Telegram-бота, добавить его в группу и получить ID группы.

### Шаг 1. Создай бота и получи токен

Этот шаг делается один раз. Дальше бот будет жить и работать сам.

**1. Открой Telegram и найди BotFather**

Нажми на строку поиска в Telegram (в мобильной версии — сверху, в десктопной — слева сверху). Введи `BotFather`. Найди в результатах пользователя с синей галочкой (верифицированный аккаунт) и именем «BotFather». Открой чат с ним.

**2. Запусти создание нового бота**

В поле ввода сообщения напиши:
```
/newbot
```
и отправь. BotFather ответит списком команд — не обращай внимания, он просто подтверждает, что команда принята.

**3. Придумай имя бота**

BotFather попросит ввести **имя** бота (name) — это то, как он будет отображаться у людей в списке контактов. Напиши что-то понятное, например:
```
Мой садовый бот
```
Или любое другое имя на русском или английском. Отправь.

**Важно:** если при вводе имени ты случайно нажал Enter с пустой строкой или написал слишком длинное имя — просто напиши `/newbot` заново.

**4. Придумай username бота**

Теперь BotFather попросит ввести **username** — уникальный адрес бота. Требования:
- должен заканчиваться на `bot` (например: `MyGardenBot` или `moy_sadoviy_bot`)
- только латинские буквы, цифры и подчёркивания
- без пробелов
- должен быть уникальным (если имя занято, BotFather скажет «Sorry, this username is already taken»)

Пример правильного username:
```
MoySadoviyBot
```
или
```
moy_sad_bot
```

Напиши свой вариант и отправь.

**5. Получи токен**

Если всё прошло успешно, BotFather пришлёт сообщение в таком виде:

```
Done! Congratulations on your new bot. You will find it at t.me/MoySadoviyBot. You can now add a description, section and profile picture for your bot, see /help for a list of commands.

Use this token to access the HTTP API:
1234567890:ABCDefghIJKlmnopQRStuVWXyz0123456789

Keep your token secure and store it safely, it can be used by anyone to control your bot.
```

**Вот это и есть токен:**
```
1234567890:ABCDefghIJKlmnopQRStuVWXyz0123456789
```

**6. Скопируй токен**

Выдели всю строку с токеном (от первой цифры до конца) и скопируй. Храни его в секрете — кто знает токен, тот может управлять твоим ботом.

Токен понадобится на шаге 4, когда будем заполнять файл `.env`.

**7. (необязательно) Настрой картинку и описание**

Можешь сразу дать боту аватарку и описание — команды `/setuserpic` и `/setdescription` в том же чате с BotFather.

### Шаг 2. Добавь бота в группу

Бот не может сам зайти в группу — его должен добавить человек.

**На мобильном телефоне (Android / iOS):**
1. Открой Telegram, нажми «✏️ Написать сообщение» (круглая кнопка снизу справа).
2. Выбери «Новая группа».
3. Выбери участников из списка — обязательно найди и добавь своего бота (поиск по username, например `@MoySadoviyBot`).
4. Добавь ещё хотя бы одного человека (Telegram требует минимум 2 участников, но после создания его можно удалить).
5. Нажми «Далее» → придумай название группы → «Создать».

**На компьютере:**
1. Нажми на три полоски ☰ слева сверху → «Создать группу» или «New Group».
2. Введи название группы.
3. В поле «Добавить участников» начни вводить username бота (например `MoySadoviyBot`), выбери его из результатов.
4. Добавь ещё одного человека (любого, потом удалишь).
5. Нажми «Создать».

**Сделай бота администратором группы:**
Бот должен быть администратором, иначе Telegram блокирует его сообщения.

1. Зайди в группу, нажми на её название вверху экрана.
2. Выбери «Управление группой» или «Group Management».
3. Раздел «Администраторы» → «Добавить администратора».
4. Выбери своего бота из списка.
5. Telegram спросит, какие права дать боту. Ничего лишнего не нужно, но **обязательно сними галочку «Анонимность»** (если бот анонимный, его сообщения нельзя отследить, и некоторые функции могут не работать).
6. Нажми «Сохранить» или «Готово».

### Шаг 3. Узнай ID группы

Теперь нужно узнать числовой ID группы — это номер, по которому Telegram понимает, куда отправлять сообщения.

**Способ 1 (самый простой):**
1. Напиши в группу любое сообщение (например «привет»). Бот его увидит.
2. Открой браузер на телефоне или компьютере.
3. В адресную строку вставь ссылку ниже, заменив `ВАШ_ТОКЕН` на тот самый токен из шага 1:

   ```
   https://api.telegram.org/bot1234567890:ABCDefghIJKlmnopQRStuVWXyz0123456789/getUpdates
   ```

4. Нажми Enter. Откроется страница с непонятным текстом — это JSON. Выглядит он примерно так:

   ```json
   {"ok":true,"result":[{"update_id":123456789, "message":{"message_id":1, "from":{...}, "chat":{"id":-1001234567890, "title":"Название группы", "type":"group"}, "text":"привет", ...}}]}
   ```

5. В этом тексте найди слово `"chat"` и рядом с ним `"id"`. Нужное значение — **число с минусом**. Например:

   ```
   "chat":{"id":-1001234567890, "title":"Название группы"
   ```

6. В данном примере ID группы: `-1001234567890`. Скопируй это число целиком (вместе с минусом).

**Если пришёл пустой ответ `{"ok":true,"result":[]}`:**
Значит, бот не видит сообщений в группе. Вернись к шагу 2 и убедись, что:
- бот добавлен в группу
- бот назначен администратором
- ты написал хоть одно сообщение в группу ПОСЛЕ того, как бот стал администратором

После этого обнови страницу в браузере — данные появятся.

**Если открывается страница с ошибкой «Not Found»:**
Проверь, что ты правильно скопировал токен. В ссылке должно быть `bot` сразу перед токеном без пробела: `bot1234567890:...`, а после токена — `/getUpdates`.

### Шаг 4. Заполни .env

Файл `.env` находится в папке проекта. Открой его любым текстовым редактором (Блокнот, VS Code и т.д.).

Сейчас в нём что-то вроде этого:
```
API_KEY=sk-d0wXpIqVhc1oJzmmuoOS6xDEsqA6h0FI
MODEL=gpt-5.5
TELEGRAM_BOT_TOKEN=your_bot_token_here
TELEGRAM_CHAT_ID=your_chat_id_here
```

Замени две нижние строчки:

1. Вместо `your_bot_token_here` вставь токен, который дал BotFather.
   Получится примерно так:
   ```
   TELEGRAM_BOT_TOKEN=1234567890:ABCDefghIJKlmnopQRStuVWXyz0123456789
   ```

2. Вместо `your_chat_id_here` вставь ID группы, который ты нашёл в JSON на шаге 3.
   Получится примерно так:
   ```
   TELEGRAM_CHAT_ID=-1001234567890
   ```

**Важно:** никаких кавычек, никаких пробелов вокруг знака `=`. Просто `НАЗВАНИЕ=значение`.

Сохрани файл и закрой редактор.

### Шаг 5. Проверь

1. Запусти проект (если он не запущен):
   ```powershell
   python app.py
   ```

2. Открой сайт: http://localhost:5000

3. Нажми на любую карточку дерева — откроется статья.

4. Нажми кнопку **«📢 Опубликовать в Telegram»**.

Если всё настроено верно:
- Кнопка на секунду станет серой, потом снова синей
- Под кнопкой появится зелёная надпись «✅ Статья опубликована в Telegram!»
- В твоей группе Telegram появится сообщение со статьёй

Если появилась **красная надпись с ошибкой** — прочитай её текст:
- «TELEGRAM_BOT_TOKEN не настроен» — ты не вставил токен в `.env`
- «TELEGRAM_CHAT_ID не настроен» — ты не вставил ID группы
- «chat not found» — бот не в группе или не администратор, проверь шаг 2
- Любая другая ошибка — скопируй её текст и поищи в Google или напиши мне

## Как это работает

1. Пользователь открывает главную страницу — видит 8 карточек с деревьями
2. Нажимает «Открыть материал» на любой карточке
3. Flask получает запрос `/article/<id>` и проверяет кэш
4. Если статьи ещё нет — шлёт запрос в ProxyAPI с промптом для генерации
5. Модель возвращает JSON с полями `intro`, `details`, `steps`, `care`, `image_placeholder`
6. Статья сохраняется в памяти (словарь `article_cache`) и отдаётся шаблону
7. При повторном заходе API не дёргается — данные берутся из кэша
8. Кнопка «📄 Сохранить в PDF» запускает html2pdf.js, который конвертирует содержимое `#article-content` в PDF-файл
9. Кнопка «📢 Опубликовать в Telegram» отправляет POST-запрос на `/publish/<id>`, Flask форматирует статью в HTML-разметку и отправляет через Telegram Bot API

## Переменные .env

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `API_KEY` | — | Ключ доступа к ProxyAPI (обязателен) |
| `MODEL` | `gpt-5.5` | Имя модели для генерации статей |
| `TELEGRAM_BOT_TOKEN` | — | Токен Telegram-бота (для публикации статей) |
| `TELEGRAM_CHAT_ID` | — | ID чата/группы Telegram, куда публиковать |

## Production

В продакшене не используйте `.env` файл — задавайте переменные окружения на уровне сервера (Render, Railway, VPS через systemd).
Замените `app.run(debug=True)` на production WSGI-сервер (gunicorn, waitress).
